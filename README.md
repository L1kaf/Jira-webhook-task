Данный вебхук предназначен для автоматической смены статуса родительской задачи на основе подзадач в jira. Работает по такой схеме:
1. Статус истории меняются на статус подзадач, если все подзадачи имеют один и тот же статус. К  примеру, все задачи в статусе "Тестирование", значит и история должна сменить статус на "Тестирование"
2. Если хотя бы одна из подзадач находится в статусе "В  работе", то  и история должна иметь статус в работе. 
3. Иначе статус истории определяется по подзадаче которая находится левее по прогрессу статусов на доске. К примеру. Если 2 подзадаче находятся в статусе "Ревью", а 1 подзадача находится в статусе "В работе", то статус истории должен быть "В работе".

В файле .env прописывается путь до логов, адрес jira и токен УЗ, от которой будут происходит переходы.
Вебхук формата: 
```
http://ip:8098/webhook?issue=${issue.key}
```
